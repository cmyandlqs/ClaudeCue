# ccCue 需求文档

## 1. 产品定位

ccCue 是面向 Windows 上 Claude Code 用户的桌面通知与回焦助手。目标是在不改变 Claude 原生终端使用方式的前提下，让用户在终端不在前台时仍能及时收到关键事件，并一键回到终端继续操作。

## 2. 产品形态

产品采用双入口、统一运行时的形态：

1. 普通用户：EXE 安装包（默认入口）。
2. 开发与运维：CLI 命令（安装、诊断、恢复、卸载）。

运行时策略已确定：安装后不依赖用户机器预装 Python/PySide6/pywin32。

## 3. 平台与架构约束

1. 仅支持 Windows 10/11。
2. Claude Code 必须保持原生终端运行，不使用 wrapper 包裹终端。
3. 事件来源必须是 Claude 官方 hooks。
4. 事件链路固定为：`hooks -> bootstrap -> notifier(/event)`。

## 4. 用户与核心场景

1. 普通用户安装后直接使用 Claude Code，任务完成/权限请求/停止时看到桌面通知。
2. 用户点击通知或使用快捷键时，能够快速回到对应终端窗口。
3. 用户出现配置异常或通知异常时，可以用诊断与恢复能力自助修复。

## 5. 功能需求

### 5.1 Hook 接入与事件标准化

1. 读取 Claude hook stdin JSON。
2. 将 Notification/Stop/PermissionRequest 等事件映射为统一事件模型。
3. Hook 执行失败不得影响 Claude 主流程（hook 进程始终安全退出）。

### 5.2 Bootstrap 与按需拉起

1. hook 触发时先检查 notifier 健康状态（`/health`）。
2. notifier 未运行时自动后台拉起。
3. 事件最终通过本地 HTTP 投递到 notifier。
4. 同一会话首次触发可展示一次轻量“已连接”提示（不刷屏）。

### 5.3 Notifier 展示层

1. 显示桌面浮层通知（支持自动消失）。
2. 系统托盘常驻管理入口。
3. 声音提醒可用。
4. 单实例运行，避免重复启动多个 notifier。

### 5.4 回焦能力

1. 点击通知可触发回焦到目标终端窗口。
2. 提供全局快捷键回焦（含冲突回退策略）。
3. 多终端并行时支持更稳健的目标识别（窗口 hint + 进程/类名/标题复核）。
4. 回焦失败时需要有可观测日志与可诊断信息。

### 5.5 配置与安全

1. 只管理 `settings.json` 中 hooks 相关目标条目。
2. 写入前必须备份。
3. 写入后必须校验，失败自动回滚。
4. 支持 baseline 恢复与按备份恢复。
5. 卸载默认恢复安装前配置，并支持彻底清理模式。

### 5.6 CLI 能力

1. 安装：写入 hooks + 部署运行时。
2. 卸载：移除管理项并恢复配置。
3. 诊断：检查 hooks、notifier 健康状态、关键运行条件。
4. 恢复：支持从最近或指定备份恢复。

### 5.7 安装与交付

1. 提供两种安装路径：
   - 普通用户路径：安装 EXE 包。
   - 开发者路径：`git clone` 后执行一条命令安装（如 `python -m cli.main install --source .`）。
2. 两种安装路径都必须走同一套配置安全与运行时校验逻辑。
3. 安装路径必须可配置：
   - 默认路径可为 `%LOCALAPPDATA%\ccCue`。
   - 用户必须可以在安装时改到其他磁盘（如 `D:\Apps\ccCue`）。
   - CLI 安装必须支持 `--target` 显式指定目标目录。
4. 需向用户公开安装体积信息（安装包大小、安装后预计占用），避免“隐式占用 C 盘”。
5. 安装完成后执行自检：
   - hooks 写入成功
   - notifier 健康端点可达
6. 卸载后不破坏用户原有 Claude 配置。

## 6. 非功能需求

1. 稳定性：异常情况下不阻塞 Claude 正常使用。
2. 可观测性：日志可定位“事件是否送达、回焦是否命中、失败在哪一步”。
3. 可恢复性：配置损坏或错误写入可回滚恢复。
4. 可维护性：模块职责清晰（hook ingestion / event normalize / notifier presentation / installer）。

## 7. 验收标准

1. 全新 Windows 机器安装后，用户可在短时间内完成首个通知闭环。
2. Claude 事件能稳定触发通知，且不影响 Claude 原流程。
3. 点击通知或快捷键可回到终端（在系统焦点策略允许范围内尽可能稳定）。
4. `doctor/restore/uninstall` 可用，异常可自助处理。
5. 卸载后配置可恢复，且无残留关键污染项。
