  一、你现在的问题本质
  你当前实现“功能已经可用”，但“产品不可持续使用”卡在两点：

  1. 用户怎么安装和升级不清晰（目前偏开发者手工操作）。
  2. notifier 需要手动 python -m notifier.main，这不符合真实使用习惯。

  ———

  二、我建议的产品形态（双轨）

  1. 面向普通用户：Windows 安装包

  - 双击安装（Next-Next-Finish）
  - 自动写 Claude hooks 配置
  - 自动放置/注册 notifier 启动入口
  - 控制面板可卸载

  2. 面向开发者：CLI 命令

  - cccue install：安装配置
  - cccue uninstall：卸载配置
  - cccue doctor：健康检查（hooks、端口、进程、权限）

  为什么要双轨：普通用户要“无脑安装”，开发者要“可调试可脚本化”。

  ———

  三、notifier 自动启动应该怎么做（重点）

  你提到“运行 claude 后自动启动 notifier”，这是对的。推荐“按需启动”，不是开机常驻。

  流程：

  1. Claude 触发 hook（Notification/Stop/...）
  2. hook 不直接发 notifier，而是先走一个很轻的 bootstrap 脚本
  3. bootstrap 做两件事：

  - 检查 http://127.0.0.1:19527/health
  - 如果 notifier 不在，后台启动 notifier（无窗口）

  4. 启动后再把本次事件转发给 notifier
  5. notifier 单实例运行（防止重复起多个）

  结果：用户不需要手动运行 main.py，只要用 Claude，就自动可用。

  ———

  四、你会得到的用户体验

  1. 第一次：安装一次即可。
  2. 日常：直接 claude，通知自动生效。
  3. 故障：cccue doctor 一键诊断问题，不用手查日志。

  ———

  五、下一步落地顺序（最实用）

  1. 先做 bootstrap + notifier 单实例（解决“手动启动”痛点）
  2. 再做 cccue doctor
  3. 最后做安装包封装（MSI/EXE）
