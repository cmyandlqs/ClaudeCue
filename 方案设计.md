# 方案设计.md

## 架构结论

明确放弃 wrapper 路线，采用 Claude Hooks 事件驱动架构。

目标链路：

`Claude hooks -> hooks/bootstrap.py -> notifier health check -> (必要时启动 notifier) -> /event`

## 模块职责

1. `hooks/`
   - 读取 stdin hook payload
   - 事件映射
   - 转发/拉起入口（bootstrap）

2. `notifier/`
   - 本地 HTTP 服务
   - 悬浮通知、托盘、声音、回焦
   - 单实例运行

3. `config/state_manager.py`
   - 安全写入 `~/.claude/settings.json`
   - 备份索引、baseline、恢复、回滚
   - doctor 检查

4. `cli/main.py`
   - install / uninstall / doctor / restore / list-backups

5. `installer/`
   - bat 入口
   - Inno Setup 脚本与编译脚本

## 配置安全策略

1. 仅修改 `hooks.Notification/Stop/PermissionRequest`。
2. 写入前强制备份。
3. 写入后结构校验与命令校验。
4. 出错自动回滚。
5. 卸载默认恢复 baseline。

## 风险与后续

1. 用户机器 Python 依赖不一致（建议后续加入运行时打包或预检安装）。
2. 历史日志/文案编码不一致（需统一 UTF-8 文案）。
