# 第一阶段开发文档

## 阶段目标

第一阶段不再实现 wrapper 监听链路，而是完成 hook 版通知 MVP 设计与重建准备。

目标是形成一条可运行的最小闭环：

```text
Claude Code hooks
  -> hook 转发脚本
  -> notifier 本地 HTTP 服务
  -> 悬浮提醒展示
```

## 本阶段范围

### A. Hook 接入

- 选择 Claude Code 可用的 hooks 事件
- 明确 stdin JSON 输入格式
- 设计 hook 脚本的标准输入输出和失败处理策略

### B. Notifier 展示层

- 常驻后台进程
- 本地 HTTP 接口
- 悬浮窗展示
- 托盘驻留
- 提示音
- 点击提醒后聚焦回 Windows Terminal

### C. 统一事件协议

统一 hook 层与 notifier 层之间的数据结构，避免展示层直接依赖 Claude 原始 payload。

### D. Windows 安装方式

- `install.bat` 或等价脚本
- 配置 `~/.claude/settings.json`
- 提供启动 notifier 的说明

## 明确不做

以下内容不再属于第一阶段实现范围：

- wrapper 启动 Claude
- stdout/stderr 抓取
- 基于终端文本规则匹配事件
- 伪终端或兼容性补丁式方案

## 建议里程碑

### M1: 协议与 hooks 配置

- 整理 Claude hooks 事件
- 确认统一事件模型
- 形成 settings.json 配置样例

### M2: Hook 转发脚本

- 从 stdin 读取 hook payload
- 映射为本地统一事件
- POST 到 notifier 本地服务

### M3: Notifier MVP

- 本地 HTTP 接口
- 悬浮提醒
- 托盘与声音

### M4: Windows 安装体验

- 一键安装脚本
- 配置落盘
- 运行说明与排错说明

## 测试策略

第一阶段测试重点不再是 CLI 包装，而是：

1. hook payload 到统一事件的映射测试
2. notifier HTTP 接口测试
3. 展示层的非 GUI 逻辑测试
4. Windows 配置脚本的手工验证
