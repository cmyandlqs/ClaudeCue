## 核心目标

为 Claude Code 开发一个 Windows 平台辅助工具，目标分为两部分：

1. 终端侧的对话历史导航能力
2. 全局置顶的桌面提醒层

当前优先级更高的是第 2 项，也就是在 Claude Code 后台运行、任务完成、等待输入、权限拦截等时机及时唤醒用户。

## 运行约束

1. 仅考虑 Windows 平台
2. 主运行环境是 Windows Terminal

## 已确认的问题

原先的 `wrapper` 方案已经确认不可行，原因如下：

1. Claude Code 是交互式 CLI，需要真实 TTY
2. `subprocess.Popen(stdout=PIPE, stderr=PIPE)` 会剥离终端交互环境
3. 在这种模式下，Claude Code 无法作为正常工作流稳定运行
4. 这不是小修小补能解决的问题，Windows 伪终端方案复杂且不适合作为当前项目主路径

## 当前结论

项目后续只走 Claude Code 原生 hooks 方案，不再继续 wrapper 路线。

旧方案：

- wrapper 启动 Claude
- 抢占 stdout/stderr
- 规则匹配
- 转发通知

新方案：

- Claude 正常在终端中启动
- Claude hooks 在关键时机触发
- hook 脚本把事件转成统一结构
- 本地 notifier 展示层接收并弹出提醒

## 新方案的理解

hook 用来“监控 Claude Code 的关键事件”，但项目仍然需要独立展示层。也就是说，整体职责和之前设想类似，只是事件来源从“wrapper 抓取输出”变成“Claude 官方 hooks 主动推送”。

推荐的职责拆分：

- `hooks`：接收 Claude Code hook 事件，做轻量转换与转发
- `notifier-app`：桌面悬浮窗、托盘、提示音、聚焦终端
- `localhost HTTP`：hooks 与 notifier 之间的本地通信接口

## 下一步实现范围

1. 新增 hooks 脚本
2. 重建 notifier 展示层
3. 设计统一事件格式
4. 提供 Windows 下的一键安装或配置脚本
5. 文档与实现全部围绕 hook 架构展开，不再保留旧 wrapper 设计
